Control Structures (Python Style)

A control structure determines the order in which statements will be executed. In addition to sequential execution, the Jeroo language supports four control structures
  • if
  • if-else
  • cascaded if (a variation on if-else)
  • pretest while


The if Structure

The if structure is used to define an optional block of code.

if condition :

    #-- true block (indented)  

#-- next statement (not indented)
  • If the condition is true, execute the "true block " exactly once,
    and continue with the "next statement".
  • If the condition is false, skip the "true block",
    and resume execution with the "next statement".
  • Do not go back and recheck the condition.
  • SYNTAX NOTES:
    • The first line of the "true block" must be indented further than the if.
    • The "true block" includes all lines that are indented at least as far as the first line of the block.
    • The "next statement" should be indented exactly as far as the if.


The if-else Structure

The if-else structure is used to define alternate blocks of code.
Only one of the blocks will be executed.

if condition :

    #-- true block (indented)

else:

    #-- false block (indented)  

#-- next statement (not indented)
  • If the condition is true, execute the "true block " exactly once,
    and resume execution with the "next statement".
  • If the condition is false, execute the "false block" exactly once,
    and resume execution with the "next statement".
  • Do not go back and recheck the condition.
  • SYNTAX NOTES:
    • The first line of the "true block" must be indented further than the if.
    • The "true block" includes all lines that are indented at least as far as the first line of the block.
    • The else: should be indented exactly as far as the if.
    • The first line of the "false block" must be indented further than the else.
    • The "false block" includes all lines that are indented at least as far as the first line of the block.
    • The "next statement" should be indented exactly as far as the if.


The cascaded if Structure

The cascaded if is a variation on the if-else structure.
The cascaded if is used to define three or more blocks of code.
Only one of the blocks will be executed.

if condition-0 : 

    #-- block-0  (indented)

elif condition-1 : 

    #-- block-1 (indented)


#-- other elif blocks go here



elif condition-n : 

    #-- block-n (indented)

else:

    #-- default block (indented)  

#-- next statement
  • Jeroo checks the conditions in the order that they appear.
  • When the first true condition is located, the corresponding block will be executed, and all other blocks and conditions will be skipped.
  • If all conditions are false, the default block will be executed.
  • After any block has been executed, execution resumes with the "next statement".
  • SYNTAX NOTES:
    • The first line of each block must be indented further than the if, else, or elif at the start of that block.
    • Each block includes all lines that are indented at least as far as the first line of the block.
    • The elif and else lines should be indented exactly as far as the if.
    • The "next statement" should be indented exactly as far as the if.


The pretest while Structure

The pretest while structure (also called a while loop) is used to define a block of code that will be executed repeatedly as long as a specified condition is true.

while condition :

    #-- loop body (indented)  

#-- next statement
  • When the condition is true, execute the "loop body " once,
    then go back and reevaluate the condition.
  • When the condition is false, skip the "loop body",
    and resume execution with the "next statement".
  • SYNTAX NOTES:
    • The first line of the loop body must be indented further than the while.
    • The loop body includes all lines that are indented at least as far as the first line of the body.
    • The "next statement" should be indented exactly as far as the while.
